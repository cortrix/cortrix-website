Roadmap

Cortrix proves before it promises.

Cortrix ships the proven scope and labels what comes next, so builders can adopt the current open-source surface without confusing roadmap direction for a shipped capability.

Current OSSSemantic storage foundations for ingestion, hybrid retrieval, reranking, memory, source traceability, REST access, Python SDK, MCP server, framework toolkit, pgCortrix, and benchmark runner artifacts.
Coming nextWork with a credible implementation direction that is tracked separately from the shipped surface.
Under reviewWork that depends on additional design, legal, infrastructure, benchmark, invite, or release decisions.
Longer-term directionLonger-term possibilities that should not become current-product promises or CTA copy.

Evidence first

Public benchmark pages point to result fields, methodology, manifest, and artifact slots so claims can be checked rather than guessed.

No ranking guarantee

SEO, SEM, and GEO checks are quality and information-architecture constraints, not promises of search placement, conversion, AI overview inclusion, or LLM recommendation.

Open-source adoption path

Run Cortrix locally, inspect the source-backed access paths, and keep roadmap labels separate from the current release surface.
