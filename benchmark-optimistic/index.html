What this page is, and what it is not

  • It is based on BEIR-style retrieval evaluation: Recall@10, NDCG@10, and latency gates.
  • It is a retrieval-quality story about reducing glue code across agent data workflows.
  • It is not an end-to-end RAG answer-quality claim.
  • It is not a claim that Cortrix already beats Qdrant, Milvus, Weaviate, pgvector, Chroma, OpenSearch, Elastic, Vespa, OpenAI, Cohere, or Voyage.
Reference note: public retrieval posts from OpenSearch, Vespa, and Elastic support the baseline categories used for ecosystem context. They are not used as same-run Cortrix comparison results, endorsements, or product rankings.
Artifact rule: each public result should stay attached to raw JSON, CSV summary, methodology notes, hardware/software details, run count, dataset list, and source snapshots.
