用一套检索系统，替代脆弱的 RAG 拼装栈。

Cortrix 在 BEIR-style 任务上按完整检索栈进行预发布预测：混合检索、reranking、enrichment、记忆上下文和来源追踪由一套系统承接，而不是把搜索、向量存储、reranker 和审计代码分散拼接。

Method boundary

这个页面是什么，不是什么

  • 它是 用于 launch-page review 和红蓝军检验的预发布视觉候选页。
  • 它基于 BEIR-style retrieval evaluation：Recall@10、NDCG@10 和 latency gates。
  • 它不是 公开实测 benchmark 结果。
  • 它不是 声称 Cortrix 已经击败 Qdrant、Milvus、Weaviate、pgvector、Chroma、OpenSearch、Elastic、Vespa、OpenAI、Cohere 或 Voyage。
Reference-note for review: OpenSearch、Vespa 和 Elastic 的公开 retrieval posts 只用于支持本页 baseline categories 的合理性。它们不是 Cortrix 的同跑比较结果、背书或产品排名。
Release rule: 页面进入 production copy 前，projection bars 必须替换为 raw JSON、CSV summary、methodology notes、hardware/software details、run count、dataset list 和 source snapshots。
