Benchmark

用可复现方法预测 BEIR 检索质量。

Cortrix 当前采用最佳预测区间做发布文案压力测试：Recall@10 约 0.85，NDCG@10 约 0.86，相对 vector-only baseline 最高提升 60%。正式发布前需要用 F44 公开运行产物替换预测值。

场景

最佳发布区间

Cortrix 预测：Recall@10 = 0.85，NDCG@10 = 0.86，相对提升最高 60%。这是第二轮红蓝军使用的最佳场景。

数据集

BEIR 五子集计划

F44 计划使用 BEIR official library 与五个公开子集：HotpotQA、FiQA、SciFact、NFCorpus、Touche。首发 benchmark 不使用自建数据集。

指标

Recall@10、NDCG@10、P50 latency

公开报告聚焦检索质量与用户可感知延迟，并记录硬件、软件、运行次数和来源快照。

来源与可比性

  • Qdrant / Milvus reference: Qdrant official benchmarks，检索日期 2026-06-02，vector-search latency / precision reference，不参与 BEIR uplift 计算。
  • Weaviate reference: Weaviate ANN benchmark，检索日期 2026-06-02，SIFT1M ANN reference，不参与 BEIR uplift 计算。
  • pgvector reference: pgvector public docs，检索日期 2026-06-02；未识别直接可比 BEIR 数字。
  • Chroma reference: Chroma public docs，检索日期 2026-06-02；未识别直接可比 BEIR 数字。

方法

  • 通过 Docker 和 BEIR official library 运行 Cortrix。
  • 使用公开 qrels 和固定 top-k 检索设置。
  • 发布原始 JSON、CSV 汇总、方法说明和复现说明。

可复现性

  • 记录 source URL、version、retrieval date、硬件、软件和运行次数。
  • 对手数据不自跑时，公开引用来源和可比性限制。
  • 不把不可比数字强行放进同一结论。

解释边界

  • 故事不是“现有数据库不好”。
  • 故事是 Agent 团队需要减少 retrieval、memory、audit 之间的胶水代码。
  • 正式发布前必须用公开运行产物替换预测值。
