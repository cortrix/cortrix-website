Benchmark

Projected BEIR retrieval quality with a reproducible release method.

Cortrix uses a best-case benchmark envelope for launch-copy testing: Recall@10 around 0.85, NDCG@10 around 0.86, and up to 60% relative lift against the planned vector-only BEIR baseline after the public F44 run replaces projections with artifacts.

Scenario

Best-case release envelope

Cortrix projection: Recall@10 = 0.85, NDCG@10 = 0.86, relative lift up to 60%. This is the best-case scenario used for the second red-blue review round.

Datasets

BEIR five-subset plan

The F44 plan uses BEIR official library and five public subsets: HotpotQA, FiQA, SciFact, NFCorpus, and Touche. No custom dataset is used for the first release benchmark.

Metrics

Recall@10, NDCG@10, P50 latency

The public report centers on retrieval quality and user-facing latency, with reproducibility notes for hardware, software, run count, and source snapshots.

Methodology

  • Run Cortrix through Docker and BEIR official library.
  • Use public qrels and fixed top-k retrieval settings.
  • Publish raw JSON, CSV summary, methodology, and reproducibility notes.

Reproducibility

  • Record source URL, version, retrieval date, hardware, software, and run count.
  • Keep public data transparent when competitors are not self-run.
  • Do not compare incomparable numbers without caveats.

Interpretation

  • The story is not that existing databases are bad.
  • The story is that agent teams need less glue code across retrieval, memory, and audit.
  • Final numbers must be replaced by public artifacts before production publication.
